Introduction to MongoDB

MongoDB is an open-source, cross-platform NoSQL document database developed by MongoDB Inc. Instead of storing data in rows and columns like relational databases, MongoDB stores data as flexible BSON (Binary JSON) documents inside collections.

🗄️

Document-Oriented

Data stored as BSON documents — rich, flexible, self-describing.

⚡

High Performance

Embedded data reduces I/O and supports fast read/write workloads.

📦

Scalable

Horizontal scaling via sharding across multiple servers.

🔄

Flexible Schema

No predefined schema — documents in a collection can differ.

MongoDB vs SQL

SQL TermMongoDB EquivalentDescription
DatabaseDatabaseContainer for collections
TableCollectionGroup of documents
RowDocumentA single record (BSON)
ColumnFieldKey-value pair in a document
Primary Key_idUnique identifier per document
JOIN$lookupCombining data from collections

Installation & Setup

Install MongoDB (Ubuntu/Debian)

bash
# Import public GPG key
wget -qO - https://www.mongodb.org/static/pgp/server-7.0.asc | sudo apt-key add -

# Add MongoDB repo
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list

# Install
sudo apt-get update
sudo apt-get install -y mongodb-org

# Start MongoDB service
sudo systemctl start mongod
sudo systemctl enable mongod

Connect via Mongo Shell

shell

mongosh

# Show all databases
show dbs

# Switch / create database
use myDatabase

# Show collections
show collections

Core Concepts

Documents

A document is a JSON-like object made of key-value pairs. MongoDB stores documents as BSON (Binary JSON) which supports additional data types.

document example
{
  "_id": ObjectId("64e1a2b3c4d5e6f7a8b9c0d1"),
  "name": "Sameeksha",
  "age": 22,
  "email": "sameeksha@example.com",
  "skills": ["MongoDB", "Node.js", "React"],
  "address": {
    "city": "Bengaluru",
    "pincode": "560001"
  },
  "createdAt": ISODate("2024-01-15T10:30:00Z")
}

ObjectId

MongoDB auto-generates a unique _id field of type ObjectId for every document. It is a 12-byte identifier containing a timestamp, machine ID, process ID, and a random counter.

💡

Tip: ObjectId embeds a creation timestamp, so you can extract when a document was created using ObjectId.getTimestamp() — no separate createdAt field needed!

BSON Data Types

  • String — UTF-8 encoded text
  • Integer — 32-bit or 64-bit signed integer
  • Double — 64-bit IEEE 754 floating point
  • Boolean — true / false
  • Date — UTC datetime stored as milliseconds since epoch
  • Array — Ordered list of values
  • Object — Embedded/nested document
  • ObjectId — 12-byte unique identifier
  • Null — Represents a null/absent value
  • Binary Data — Used for images, files, etc.
  • Decimal128 — High-precision decimal (for financial data)

CRUD Operations

Create — Insert Documents

mongosh
// Insert one document
db.students.insertOne({
  name: "Sameeksha",
  branch: "Computer Science",
  year: 3
})

// Insert multiple documents
db.students.insertMany([
  { name: "Arjun", year: 2 },
  { name: "Priya", year: 4 }
])

Read — Find Documents

mongosh
// Find all documents
db.students.find()

// Find with filter
db.students.find({ year: 3 })

// Find one document
db.students.findOne({ name: "Sameeksha" })

// Projection (select specific fields)
db.students.find({}, { name: 1, branch: 1, _id: 0 })

// Sort, Limit, Skip
db.students.find().sort({ name: 1 }).limit(5).skip(10)

Update — Modify Documents

mongosh
// Update one document
db.students.updateOne(
  { name: "Sameeksha" },
  { $set: { year: 4 } }
)

// Update many documents
db.students.updateMany(
  { year: 1 },
  { $set: { status: "fresher" } }
)

// Increment a field
db.students.updateOne(
  { name: "Sameeksha" },
  { $inc: { score: 10 } }
)

// Upsert: insert if not found
db.students.updateOne(
  { name: "Vikram" },
  { $set: { year: 2 } },
  { upsert: true }
)

Delete — Remove Documents

mongosh
// Delete one document
db.students.deleteOne({ name: "Arjun" })

// Delete many documents
db.students.deleteMany({ year: { $lt: 1 } })

// Drop entire collection
db.students.drop()

// Drop entire database
db.dropDatabase()

Query Operators

Comparison Operators

OperatorMeaningExample
$eqEqual{age: {$eq: 20}}
$neNot equal{age: {$ne: 20}}
$gtGreater than{age: {$gt: 18}}
$gteGreater or equal{age: {$gte: 18}}
$ltLess than{age: {$lt: 30}}
$lteLess or equal{age: {$lte: 30}}
$inIn array{age: {$in: [20,21,22]}}
$ninNot in array{age: {$nin: [20,21]}}

Logical Operators

mongosh
// $and — both conditions must be true
db.students.find({
  $and: [{ year: 3 }, { branch: "CS" }]
})

// $or — at least one must be true
db.students.find({
  $or: [{ year: 1 }, { year: 4 }]
})

// $not — negates condition
db.students.find({
  year: { $not: { $gte: 3 } }
})

// $nor — none must be true
db.students.find({
  $nor: [{ year: 1 }, { year: 2 }]
})

Element Operators

mongosh
// $exists — field exists in document
db.students.find({ email: { $exists: true } })

// $type — field is of specified BSON type
db.students.find({ age: { $type: "int" } })

// $regex — pattern matching
db.students.find({ name: { $regex: /^Sam/i } })

Array Operators

mongosh
// $all — array contains all specified values
db.students.find({ skills: { $all: ["MongoDB", "React"] } })

// $size — array has exactly n elements
db.students.find({ skills: { $size: 3 } })

// $elemMatch — at least one element matches all criteria
db.orders.find({
  items: {
    $elemMatch: { price: { $gt: 100 }, qty: { $lt: 5 } }
  }
})

Aggregation Pipeline

The aggregation pipeline processes documents through a series of stages, each transforming the data. It is MongoDB's equivalent of SQL GROUP BY and complex queries.

Pipeline Stages Overview

StagePurpose
$matchFilter documents (like WHERE)
$groupGroup and aggregate (like GROUP BY)
$projectReshape documents (select/add/remove fields)
$sortSort results
$limitLimit number of results
$skipSkip N documents
$unwindDeconstruct arrays into separate documents
$lookupJoin with another collection
$addFieldsAdd computed fields
$countCount documents in pipeline

Example: Group & Count

mongosh
db.students.aggregate([
  // Stage 1: filter year 3 and above
  { $match: { year: { $gte: 3 } } },

  // Stage 2: group by branch, count students
  { $group: {
    _id: "$branch",
    totalStudents: { $sum: 1 },
    avgScore: { $avg: "$score" }
  } },

  // Stage 3: sort by total students descending
  { $sort: { totalStudents: -1 } },

  // Stage 4: project output fields
  { $project: {
    branch: "$_id",
    totalStudents: 1,
    avgScore: { $round: ["$avgScore", 2] },
    _id: 0
  } }
])

$lookup (JOIN)

mongosh
db.orders.aggregate([
  {
    $lookup: {
      from: "customers",     // collection to join
      localField: "customerId",  // field in orders
      foreignField: "_id",        // field in customers
      as: "customerInfo"         // output array field
    }
  }
])

Indexes

Indexes improve query performance by allowing MongoDB to find documents without scanning every document in a collection.

⚠️

Trade-off: Indexes speed up reads but slow down writes (insert/update/delete) because indexes must be updated too. Create indexes only on fields you query frequently.

Creating Indexes

mongosh
// Single field index (1 = ascending, -1 = descending)
db.students.createIndex({ name: 1 })

// Compound index
db.students.createIndex({ year: 1, branch: 1 })

// Unique index
db.students.createIndex({ email: 1 }, { unique: true })

// Text index (for full-text search)
db.articles.createIndex({ content: "text" })

// TTL index (auto-expire after seconds)
db.sessions.createIndex({ createdAt: 1 }, { expireAfterSeconds: 3600 })

// List all indexes
db.students.getIndexes()

// Drop an index
db.students.dropIndex("name_1")

Explain Query (Check Index Usage)

mongosh
db.students.find({ name: "Sameeksha" }).explain("executionStats")
// Look for "IXSCAN" (index scan) vs "COLLSCAN" (collection scan)

Schema Design

Embedding vs Referencing

📎

Embedding (Denormalization)

Store related data within the same document. Best for data that is always read together and doesn't change frequently.

🔗

Referencing (Normalization)

Store a reference (ObjectId) to another document. Best for large sub-documents accessed independently.

embedding example
// Blog post with embedded comments
{
  title: "MongoDB Notes by Sameeksha",
  content: "...",
  comments: [
    { user: "Alice", text: "Great post!", date: ISODate("2024-01-01") },
    { user: "Bob",   text: "Very helpful", date: ISODate("2024-01-02") }
  ]
}
referencing example
// Order references customer by _id
{
  orderId: "ORD-1001",
  customerId: ObjectId("64e1a2b3c4d5e6f7a8b9c0d1"),
  total: 4500,
  items: [{ product: "Laptop", qty: 1, price: 4500 }]
}

Relationships

One-to-One

Embed the related document directly. Example: a user and their profile settings.

One-to-Many

Use an array of embedded documents (if small) or store the parent's _id in the child (if large). Example: one author → many books.

Many-to-Many

Store arrays of references in both documents or use an intermediary collection. Example: students ↔ courses.

many-to-many via arrays
// Student document
{ _id: ObjectId("s1"), name: "Sameeksha", courseIds: [ObjectId("c1"), ObjectId("c2")] }

// Course document
{ _id: ObjectId("c1"), title: "MongoDB", studentIds: [ObjectId("s1"), ObjectId("s3")] }

Mongoose ODM

Mongoose is an Object Data Modeling (ODM) library for MongoDB and Node.js. It provides schema validation, middleware, and a rich query API.

Install & Connect

bash
npm install mongoose
javascript
const mongoose = require('mongoose')

mongoose.connect('mongodb://localhost:27017/mydb')
  .then(() => console.log('Connected to MongoDB'))
  .catch(err => console.error(err))

Define a Schema & Model

javascript
const studentSchema = new mongoose.Schema({
  name: { type: String, required: true, trim: true },
  email: { type: String, required: true, unique: true },
  year: { type: Number, min: 1, max: 4 },
  skills: [String],
  createdAt: { type: Date, default: Date.now }
})

const Student = mongoose.model('Student', studentSchema)

CRUD with Mongoose

javascript
// Create
const s = new Student({ name: "Sameeksha", email: "s@mail.com", year: 3 })
await s.save()

// Read
const students = await Student.find({ year: 3 })
const one = await Student.findById(id)

// Update
await Student.findByIdAndUpdate(id, { year: 4 }, { new: true })

// Delete
await Student.findByIdAndDelete(id)

Mongoose Middleware (Hooks)

javascript
// Pre-save hook — hash password before saving
studentSchema.pre('save', async function(next) {
  if (this.isModified('password')) {
    this.password = await bcrypt.hash(this.password, 10)
  }
  next()
})

Transactions

MongoDB supports multi-document ACID transactions (since v4.0 for replica sets, v4.2 for sharded clusters). Use transactions when multiple documents must be updated atomically.

javascript (Mongoose)
const session = await mongoose.startSession()
session.startTransaction()

try {
  // Debit from account A
  await Account.findByIdAndUpdate(
    accountA, { $inc: { balance: -1000 } }, { session }
  )

  // Credit to account B
  await Account.findByIdAndUpdate(
    accountB, { $inc: { balance: 1000 } }, { session }
  )

  await session.commitTransaction()
  console.log("Transaction successful")
} catch (err) {
  await session.abortTransaction()
  console.error("Transaction aborted:", err)
} finally {
  session.endSession()
}

Security & Best Practices

Authentication

mongosh
// Create an admin user
use admin
db.createUser({
  user: "sameeksha_admin",
  pwd: "securePass123!",
  roles: [{ role: "userAdminAnyDatabase", db: "admin" }]
})

// Create a read-only user for a specific db
use myDatabase
db.createUser({
  user: "readUser",
  pwd: "readPass",
  roles: [{ role: "read", db: "myDatabase" }]
})

Best Practices

  • Enable authentication — Always set security.authorization: enabled in mongod.conf
  • Principle of least privilege — Grant only the permissions each user needs
  • Use TLS/SSL — Encrypt data in transit between client and server
  • Validate inputs — Use JSON Schema validation to enforce document structure
  • Regular backups — Use mongodump / mongorestore or MongoDB Atlas snapshots
  • Limit exposed ports — Never expose port 27017 to the public internet without a firewall
  • Use connection pooling — Mongoose handles this automatically
  • Monitor with Atlas / Ops Manager — Track slow queries, index usage, and replication lag
  • Avoid $where operator — It executes JavaScript and is a security risk
  • Index wisely — Use explain() to identify missing or unused indexes

Backup & Restore

bash
# Backup a database
mongodump --db myDatabase --out /backup/

# Restore a database
mongorestore --db myDatabase /backup/myDatabase/

# Export to JSON
mongoexport --db myDatabase --collection students --out students.json

# Import from JSON
mongoimport --db myDatabase --collection students --file students.json
